# 运单列表筛选与列配置设计方案

## 背景

运单表字段会越来越多，包括基础信息、渠道承运商、时间节点、轨迹状态、异常状态、财务状态、分组信息等。很多字段都有筛选价值，但如果全部放在列表顶部，会导致筛选区过长、页面拥挤、运营难以快速定位常用条件。

因此运单列表不应继续做“一整条筛选表单”，而应改为分层筛选：

```text
顶部轻筛选 = 高频条件，快速操作
高级筛选 = 低频或复杂条件，集中收纳
筛选摘要 = 显示当前已生效条件
保存视图 = 复用复杂筛选和列配置
列设置 = 控制表格显示字段
```

## 设计目标

1. 顶部只保留最高频筛选，保证运单列表第一屏清爽。
2. 支持大部分字段筛选，但通过高级筛选抽屉分组收纳。
3. 用户可以清楚看到当前生效的筛选条件，并能单独移除。
4. 支持保存常用筛选视图，减少重复操作。
5. 支持列显示配置，避免表格字段越来越多后不可读。

## 页面结构

建议运单列表顶部结构：

```text
关键词搜索
状态
渠道
客户
时间口径 + 时间范围
高级筛选
重置
导出
列设置
```

下方展示筛选摘要：

```text
已筛选：渠道 = SEA_TRUCK_US_WEST、ATD = 2026-06-01 ~ 2026-06-30、未签收、即将超时
```

再下方是运单表格。

## 顶部轻筛选

顶部只放每天最常用的条件。

建议字段：

| 筛选项 | 说明 |
| --- | --- |
| 关键词 | 运单号、柜号、提单号、货件号、客户编号、供应商等模糊搜索 |
| 状态 | 运单状态，例如在途、已到港、已签收、异常 |
| 渠道 | `channel_code` |
| 客户 | 客户名称或客户编号 |
| 时间口径 | 决定时间范围筛哪个字段 |
| 时间范围 | 与时间口径配合使用 |

时间口径建议：

```text
创建时间
ETD
ATD
ETA
ATA
预计送仓时间
签收时间
更新时间
```

这样只需要一组日期范围控件，就能筛选多个时间字段，避免页面上出现多组日期选择器。

## 高级筛选抽屉

点击 `高级筛选` 后从右侧打开抽屉。抽屉内按业务分组，不按数据库字段平铺。

### 基础信息

```text
运单号
客户
客户编号
国家 / 地区
目的港
仓库 / Amazon FC
地址关键词
渠道
承运商
船名航次
```

### 时间节点

```text
ETD 范围
ATD 范围
ETA 范围
ATA 范围
预计送仓时间范围
签收时间范围
创建时间范围
更新时间范围
```

说明：

- 顶部轻筛选使用“时间口径 + 时间范围”。
- 高级筛选中保留独立时间字段，满足复杂组合场景，例如 `ATD 在 6 月` 且 `未签收` 且 `ETA 已过`。


财务字段是否首版实现，取决于当前数据完整度。可以先在设计中预留。

## 筛选摘要条

当用户设置筛选条件后，在表格上方显示摘要标签。

示例：

```text
渠道 = SEA_TRUCK_US_WEST
客户 = ABC
ATD：2026-06-01 ~ 2026-06-30
未签收
即将超时
```

交互：

```text
点击标签上的关闭按钮：移除该条件
点击“清空”：清空全部筛选
点击某个标签：可选，回到对应筛选控件
```

这样即使筛选条件来自高级筛选抽屉，用户也能在列表页一眼看到当前条件。

## 保存视图

字段越来越多后，保存视图会比继续堆筛选项更重要。

保存视图包含：

```text
筛选条件
排序方式
列显示配置
分页大小
```

常用视图示例：

```text
今日待处理
即将超时
已超时未签收
已到港未签收
本月已签收
待签收时间审批
某客户未签收
```

视图类型：

| 类型 | 说明 |
| --- | --- |
| 系统视图 | 系统内置，所有人可见 |
| 个人视图 | 用户自己保存 |
| 团队视图 | 后续扩展，团队共享 |

首版可以先做系统视图 + 本地个人视图；后续再做服务端保存。

## 列设置

筛选字段变多时，表格列也会变多，因此需要列设置。

默认列建议：

```text

状态
客户
渠道
派送仓库
邮编
承运商
内部轨迹
承运商轨迹
供应商
货件号
亚马逊预约号

```

列设置功能：

```text
勾选显示字段
拖拽调整顺序
恢复默认列
保存为当前视图列配置
```

字段分组：

```text
基础信息
时间节点
轨迹状态
异常审批
财务业务
```

## 推荐首版范围

第一版建议先实现：

1. 顶部轻筛选：关键词、状态、渠道、客户、时间口径、时间范围。
2. 高级筛选抽屉：基础信息、时间节点、异常与审批。
3. 筛选摘要标签：展示和删除已生效条件。
4. 列设置：勾选显示字段、恢复默认。
5. 系统内置视图：即将超时、已超时未签收、已到港未签收、待签收时间审批。

暂不强制实现：

```text
拖拽列排序
团队共享视图
服务端保存个人视图
复杂财务筛选
高级条件组合 AND / OR
```

## 落地顺序

建议按以下顺序实现：

1. 梳理运单列表可筛选字段，统一前后端筛选参数命名。
2. 将顶部筛选缩减为轻筛选。
3. 增加高级筛选抽屉，并把低频字段迁入抽屉。
4. 增加筛选摘要标签。
5. 增加列设置。
6. 增加系统内置视图。
7. 根据使用情况决定是否做个人视图保存。

## 设计原则

1. 高频条件直接露出，低频条件收进抽屉。
2. 时间字段通过“时间口径 + 时间范围”复用，避免多个日期框堆在顶部。
3. 筛选结果必须可见，不能让条件藏在抽屉里。
4. 表格列和筛选条件分开治理，字段多不代表都要显示在表格里。
5. 保存视图用于承载复杂工作流，不靠顶部筛选区无限扩张。
